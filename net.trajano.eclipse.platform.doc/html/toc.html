Platform Extensions
by: Archimedes Trajano ( www.trajano.net)

This plug-in adds the following extensions to the Eclipse environment:

Copy Name To Clipboard > File Name

This copies the file name of the resource to the clipboard. It copies the OS path using resource.getLocation().toOSString() to the clipboard using new  Clipboard(...).setContents(...). To get the current display use Display.getCurrent() .
Copy Name To Clipboard > Fully Qualified Name

This copies the fully qualified name of a Java element. If a package is selected it will give the fully qualified name of the package. This will only work on Java Classes and packages.
Open Folder > In Command Prompt

This opens the containing folder of the resource or the folder itself in the command prompt window.
Open Folder > In Explorer

This opens the containing folder of the resource or the folder itself in Windows explorer.

This is similar to the functionality of EasyExplore, but allows multiple selections to be opened and logic to prevent it from opening those it cannot open.
Open Folder > In Navigator

This opens the containing folder of the resource or the folder itself in the navigator view. It will open up the view in the current perspective if needed.
Restart Workbench
(in the File menu)

Restarts the workbench. This is useful if you play around with plugins a lot which require Eclipse to restart. That way you don't have to wait for Eclipse to fully exit before you restart. This would free up more memory than just performing a garbage collection call as all the loaded plugins are freed up from memory again.

For plug-in developers, you can use this plug-in to restart your runtime workbench and it will force your plug-ins to be reloaded as well.

This is not available in the toolbar to prevent accidental restarts.

The action calls the restart() method in the current workbench.
Run Garbage Collection
(in the File menu)

This executes System.gc() which forces the JVM garbage collector to start. Please note if there is a large memory chunk to be freed, it may crash Eclipse which is why the action asks you to save your open editors before performing it (this setting is configurable).

The plug-in takes advantage of the 3.0 SDK's new background job API for garbage collection. However, it will still work with the 2.1 SDK as it automatically detects if you are using the 3.0 SDK or higher, so you will not have troubles running it on previous versions of Eclipse or WebSphere Application Developer 5.1.
