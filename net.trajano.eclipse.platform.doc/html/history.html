Release History

4.0.0
  • Build process has been converted to Tycho.
  • Minimum JDK is 1.5.
3.6.1.1
  • Corrected the logic for the fully qualified path. It was not as simple as changing the separator character if I needed support for non-Java files.
  • Added contribution to list adapters for a selected object and copy it to the clipboard.
3.6.1.0
  • Added capability of copying the fully qualified path. Useful for those instances when you use Class.getResource or Spring.
  • Added contributions to the welcome overview and what's new pages
3.5.0.1
  • Separated the selection processor plug-in into two plugins. Known overkill, but it will allow creating a feature that does not require JDT for those that may need it.
  • Fixed a bug on the OpenFolder plug-in for the Mac where it does not open the folder in command prompt when a file is selected.
  • Changed the icons and text for Open Folder to be Mac specific.
  • Added a new plug-in for Eclipse Platform property testing.
3.5.0.0
  • Target Eclipse 3.5 and above, but since I didn't really write any code specific to Eclipse 3.5 I am putting the version to be at least 3.0 and above if anyone is having trouble please file a bug report so I can determine the minimum version.
  • I am still keeping JRE 1.4 as the minimum requirement rather than taking advantage of the JRE 5.0 features for now.
  • I also took out the locale requirements since not everyone is "en" and I didn't realize it was going to cause issues.
  • Added defaults for MacOSX.
  • For MacOSX, "Open in Command Prompt" is not configurable, it works by executing an AppleScript that is hard coded into the plug-in.
3.3.0.1
  • Got rid of "Open Folder -< In Navigator" since the functionality is already available in Eclipse (somewhat) it is called "Show In...". Theoretically it might be better to put the Open Folder functionality in the same sub-menu, but Eclipse didn't expose their APIs to allow us to do it without touching the internals.
  • Migrated from org.eclipse.ui.popupMenus and switch to org.eclipse.ui.commands and org.eclipse.ui.menus. This switch will eventually enable me to use dynamic menus so users can add an arbitrary amount of Open Folder commands in the future. This is the reason why I didn't get rid of the Actions as of yet since I would be working them into actions later. This was also done as well for Copy to Clipboard.
  • The changes made the "Open Folder" also prevent "Open Folder" from being displayed if the object does not support it.
  • Changed the documentation to use the MANIFEST.MF approach
  • Added a separate field for the case when a file has been selected and Open In Explorer had been selected.
3.3.0.0
  • Migrated to use Eclipse 3.3 APIs. This is not compatible with 3.2 and below.
  • This is targeting JDK 1.4 so there are no use of advanced JDK 5.0 features such as generics in order to preserve compatibility with users still using 1.4 JDK as their VM.
  • Maven use was removed since the PDE removes the .classpath entries put in by Maven. However, the Maven style paths have been kept.
  • Some i18n work has been done. It is primarily used for establishing the environment, so the translations are going to need a lot of work. The translations are separated into fragments ending in .i18n.
  • The preference page for the recycle workspace and garbage collection has been separated into two individual pages. Since Eclipse has support for "searching" the preference pages, it would make more sense that they only contain what the user would ask for. However, each preference page description has keyword references to "Eclipse Platform Extensions" and "Platform Extensions" so the user can easily search for it using the old name.
3.2.0.4
  • Added back the "Startup" class in order for the autobuild to put in the proper state.
  • Combined the Autobuild with the Garbage collection.
3.2.0.3
  • Added back Autobuild toolbar button. It just reuses the Build Automatically action already present in the Project menu.
3.2.0.2
  • Fixed a missed <tr> in the release history page.
  • Updated help to reflect new location of Restart and GC.
  • Removed all deprecated code warnings.
  • Ran code through CheckStyle (with customized settings).
  • Changed license to Academic Free License
  • Updated Trove categorizations in the SourceForge project, but the UNIX name is still kept.
  • Made a mistake in the JRE used to compile. It now uses the IBM J2SE 1.4 SDK so it will work properly with Rational Application Developer 6.0
  • Tested with Eclipse 3.2M5 as well.
3.2.0.1
  • Applied patch 1231468.
  • Fixed problem with copying fully qualified name on objects that have no package.
  • Added support for multiple selection on copying file name and fully qualified name to clipboard. Feature Request 1251055.
  • Removed Platform menu and moved the Restart and GC to the file menu.
3.2.0.0
  • Tested and developed on Eclipse 3.2M4. However, there is no use of 3.2 specific APIs.
  • Split up the "Open Folder in..." functionality to its own plugin and is a total rewrite.
  • The marker is now {} rather than {0}.
  • The "Open Folder in..." plugin also defines an extension point for supporting other selection processors so people can create their own processor as needed without waiting for me to develop it. To facilitate this, the popup will accept any IAdaptable. However, there isn't any documentation to allow explicit activation of an IObjectActionDelegate so if the user tries to open the folder of an unsupported object, it will display a the following message: "The chosen operation is not enabled" the first time it is activated and the object type is not supported.
  • The open in navigator function is a bit smarter in that it will only be present for IResource objects.
  • A "Test" button is provided so the user can test if it the application name they entered is correct.
  • There is support for OS dependent command line strings. However, only Windows has been put in until suitable values for other OSes can be determined.
3.1.0.2
  • Added System.runFinalizer() on garbage collection.
  • Added CVS patch.
3.1.0.1
  • Fixed to work with the 3.1 M4 and above.
1.2.4
  • Removed build automatically function since Eclipse 3.0 already provides the functionality.

1.2.1

  • Fixed the formatting and icons on some of the help files.
  • Ran plugin migration utilities to migrate to Eclipse 3.0.

1.2.0

  • Open folder in Explorer and Command Prompt are run as a background task (only on Eclipse SDK 3.0 and above)
  • Restart Workbench defaults to "No"
  • Added "Copy Name To Clipboard" which copies the OS path name of the selected resource to the clipboard
  • Added "Open Folder" -> "In Navigator" which opens the containing folder in the Navigator view.
  • Added instructions on how to activate the extension on Eclipse 2.1 and WebSphere Application Developer 5.1

There were two interim releases before 1.2.0: 1.1.1 and 1.1.2, but they were removed because of a bug that prevented it from working in Eclipse 2.1 and WSAD 5.1 which I couldn't fix without a lot of work so I removed them from the SourceForge site. This release has been fixed to work on Eclipse 2.1 though.
